이를 AI 모델에 적용하는 과정에서 기술적 사고력과 문제 해결 능력을 강화했습니다. 저는 LLM 기술과 데이터 분석을 결합한 지능형 솔루션 개발을 통해 제품 고도화, 비즈니스 인사이트 발굴, 고객 맞춤형 최적화 전략 수립에 기여하며, 지속적인 학습과 팀 협업을 통해 실질적인 가치를 창출하는 AI 개발자로 성장하겠습니다.

프로젝트 경험

SUMMARIX - AI 기반 경제 뉴스 분석 웹서비스

2025.09 - 2025.10
기획 총괄 · AI 챗봇 개발

프로젝트 개요

경제 뉴스를 효율적으로 분석하고 핵심 이슈와 감성 흐름을 한눈에 파악할 수 있는 AI 기반 플랫폼입니다.
GPT-5와 RAG 기술을 결합하여 실시간 경제 질의응답이 가능한 챗봇을 구현했습니다.

시연 영상 및 GitHub

주요 담당 업무

  • 전체 시스템 아키텍처 설계 및 FastAPI 기반 백엔드 서버 구축
  • GPT-5 Function Calling을 활용한 AI 챗봇 개발 (실시간 경제지표 조회, RAG 기반 문서 검색)
  • MongoDB Atlas 연동으로 최신 뉴스 데이터 실시간 수집 및 관리
  • ECOS(한국은행), FRED(미국 연준), yFinance API 통합하여 경제지표 자동 조회 시스템 구현
  • CLOVA STT와 Google Cloud TTS 연동으로 음성 대화 기능 구현
  • Vector Store 기반 RAG 시스템 구축 및 자동 문서 갱신 파이프라인 개발
  • 팀 협업 관리 및 일정 조율, 최종 발표 준비
1단계: 데이터 수집
① APScheduler가 10분 간격으로 다양한 소스에서 경제 데이터를 자동 수집
② Naver 뉴스 크롤링을 통해 최신 경제 뉴스를 스크래핑, 한국은행 ECOS API로 국내 경제지표를 수집.
③ 미국 경제지표는 FRED API를 통해 연방기금금리를 조회, yFinance API로 주요지표, 환율 등 실시간 시세 데이터를 수집.
↓
2단계: 데이터 저장
① 수집된 모든 데이터는 MongoDB Atlas의 chatbot_rag 데이터베이스에 중앙 집중식으로 저장.
② 뉴스 기사는 제목, URL, 발행일시와 함께 저장되며, 내림차순 인덱스가 적용되어 최신 데이터 조회 성능을 최적화.
↓
3단계: AI 챗봇 엔진
① GPT-5 모델을 활용한 Function Calling 메커니즘이 사용자 질의를 분석하여 적절한 함수를 자동 선택.
② RAG 시스템은 폴더를 실시간 감시하여 파일 변경을 감지하면 자동으로 OpenAI API로 업로드하고, 이를 Vector Store에 연결.
↓
4단계: 웹 서비스 통합
① FastAPI 서버(포트 8000)가 /chat, /api/stt, /api/tts 엔드포인트를 제공.
② Spring Boot 서버(포트 8081)의 ChatController.java가 클라이언트 요청을 FastAPI로 전달하고 응답을 반환.
③ STT 기능은 CLOVA API를 사용하며, 업로드된 오디오 파일을 ffmpeg로 WAV로 변환한 후 텍스트로 변환.
④ TTS 기능은 Google Cloud TTS API로 텍스트를 MP3, WAV 형식의 음성으로 합성.
↓
6단계: 사용자 인터페이스
① Spring Boot와 Thymeleaf 템플릿 엔진이 동적 웹페이지를 렌더링.
② 사용자는 챗봇 인터페이스를 통해 텍스트 또는 음성으로 질문하고, AI가 실시간 데이터를 기반으로 답변.

핵심 기술 구현

  • Function Calling 구조: get_latest_news(), get_market(), search_docs() 등 함수를 통해 실시간 데이터 조회
  • RAG 시스템: watcher.py를 통한 문서 자동 감시 및 Vector Store 실시간 갱신
  • API 엔드포인트: /chat, /tts, /stt, /rag/docs 등 REST API 설계

프로젝트 성과

  • 실시간 경제 데이터 통합 조회 시스템 구축으로 사용자 편의성 향상
  • RAG 기반 문서 검색으로 챗봇 응답 정확도 개선
  • 음성 인터페이스 구현으로 접근성 확대
  • 팀 프로젝트 리딩 경험 및 백엔드 아키텍처 설계 역량 강화
